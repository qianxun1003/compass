# data 目录

## admission_score_model.json（合格实绩分析成果）

这就是「每个学校每一门科目要达到多少分」的**分析成果文件**。由 `scripts/analyze_admission_scores.py` 从 **合格实绩.xlsx** 分析生成，用于成绩匹配页各科阈值，以及**各大学分数要求**说明页（琥珀色按钮）的展示。

- **生成方式**：在项目根目录执行  
  `python3 scripts/analyze_admission_scores.py`
- **Excel 要求**：
  - 工作表名：`2024文科`、`2024理科`、`2023`、`2022`（表名用于年份与文理）
  - 表头列（可兼容别名）：结果、大学、学部、文理、日语、数学1、数学2、综合、物理、化学、生物、托福
  - 仅分析「合格」样本；含「不合格」「没有出愿」「不考了」等字样的行会被过滤
- **年份权重**：2024=1.0，2023=0.8，2022=0.6
- **结构**：`bunka` / `rika` → 学校名 → 学部名 → `subjects`（科目键：日语、数学1、数学2、综合、物理、化学、生物、托福）→ 每科 `min`、`p25`、`p50`、`p75`、`n`

若 Excel 尚无数据或未生成，成绩匹配页会回退到各校的 `recommendJP` / `recommendEN` 做日语与托福的匹配。
